Bienvenido a Flores Victoria Enterprise - tu plataforma completa de e-commerce con capacidades de automatización enterprise.

🔧 Instalación Rápida

bash 
# Clonar el repositorio
git clone https://github.com/empresa/flores-victoria.git
cd flores-victoria

# Instalar dependencias
npm install

# Configurar variables de entorno
cp .env.example .env

# Iniciar todos los servicios
./start-all.sh
✅ ¡Listo! El sistema estará disponible en http://localhost:3020

API RESTful completa para integrar con sistemas externos y aplicaciones móviles.

🔐 Autenticación

Todas las llamadas a la API requieren autenticación mediante JWT tokens.

http 
POST /api/auth/login
Content-Type: application/json

{
  "email": "admin@floresvictoria.com",
  "password": "tu_password"
}

Response:
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 1,
    "name": "Admin User",
    "role": "admin"
  }
}

🚀 Deployment

Guía completa para desplegar el sistema en producción usando Docker y orquestación con Docker Compose.

🐳 Docker Deployment

El sistema incluye configuraciones Docker pre-configuradas para diferentes entornos:

bash 
# Producción
docker-compose -f docker-compose.prod.yml up -d

# Staging
docker-compose -f docker-compose.staging.yml up -d

# Desarrollo
docker-compose -f docker-compose.dev.yml up -d

🔧 Variables de Entorno

env 
# Base de datos
DATABASE_URL=postgresql://user:password@localhost:5432/flores_victoria
REDIS_URL=redis://localhost:6379

# Servicios externos
SENDGRID_API_KEY=tu_api_key_de_sendgrid
PAYPAL_CLIENT_ID=tu_client_id_de_paypal
PAYPAL_CLIENT_SECRET=tu_client_secret_de_paypal

# Configuración de la aplicación
NODE_ENV=production
PORT=3020
JWT_SECRET=tu_jwt_secret_muy_seguro

# Analytics
GOOGLE_ANALYTICS_ID=GA_TRACKING_ID

⚖️ Load Balancing

Para entornos de alta disponibilidad, se recomienda usar Nginx como reverse proxy:

nginx 
upstream flores_victoria {
    server 127.0.0.1:3020;
    server 127.0.0.1:3021;
    server 127.0.0.1:3022;
}

server {
    listen 80;
    server_name floresvictoria.com;
    
    location / {
        proxy_pass http://flores_victoria;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
⚠️ Importante: Asegúrate de configurar SSL/TLS para producción usando Let's Encrypt o certificados propios.

🔧 Troubleshooting

Soluciones a problemas comunes y guías de diagnóstico del sistema.

🚨 Problemas Comunes

❌ El sitio no carga

Verificar que todos los servicios estén ejecutándose:

bash 
# Verificar estado de servicios
./check-detailed-status.sh

# Reiniciar todos los servicios
./stop-all.sh && ./start-all.sh

# Ver logs del sistema
docker-compose logs -f

🗄️ Problemas de Base de Datos

bash 
# Verificar conexión a PostgreSQL
docker exec -it postgres_container psql -U username -d flores_victoria

# Ejecutar migraciones
npm run migrate

# Backup de la base de datos
docker exec postgres_container pg_dump -U username flores_victoria > backup.sql

🔍 Diagnóstico de Performance

bash 
# Monitorear uso de recursos
docker stats

# Verificar memoria y CPU
htop

# Analizar logs de performance
tail -f logs/performance.log | grep "slow query"
