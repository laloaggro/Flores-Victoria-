🚀 Guía Rápida

💡 Nota: Esta guía te permite tener el proyecto funcionando en menos de 10 minutos.

1. Instalación

bash 
# Clonar el repositorio
git clone https://github.com/laloaggro/Flores-Victoria-.git
cd Flores-Victoria-

# Instalar dependencias
npm install

# Configurar variables de entorno
cp .env.example .env
# Edita .env con tus valores

2. Iniciar Servicios

bash 
# Iniciar todos los servicios en modo desarrollo
npm run dev:all

# O iniciar servicios individualmente:
npm run dev:gateway      # API Gateway (puerto 3000)
npm run dev:auth         # Auth Service (puerto 3001)
npm run dev:payment      # Payment Service (puerto 3003)
npm run dev:admin        # Admin Panel (puerto 3021)

3. Verificar Instalación

bash 
# Verificar puertos en uso
npm run ports:check:dev

# Validar enlaces
npm run links:validate

# Ejecutar tests
npm test

✅ ¡Listo!

Accede a las siguientes URLs:

  • API Gateway: http://localhost:3000
  • Admin Panel: http://localhost:3021
  • Frontend: http://localhost:8080

🏗️ Arquitectura

Componentes Principales

🌐 API Gateway (Puerto 3000)

Responsabilidades:

  • Punto de entrada único para todas las peticiones
  • Enrutamiento a microservicios
  • Rate limiting y autenticación
  • Logging centralizado

Tecnologías: Express.js, http-proxy-middleware, express-rate-limit

🔐 Auth Service (Puerto 3001)

Responsabilidades:

  • Autenticación de usuarios (login/register)
  • Generación y validación de JWT tokens
  • Gestión de sesiones
  • Hash de contraseñas con bcrypt

Tecnologías: Express.js, bcrypt, jsonwebtoken

💳 Payment Service (Puerto 3003)

Responsabilidades:

  • Procesamiento de pagos
  • Integración con Stripe/PayPal
  • Webhook handlers
  • Gestión de transacciones

Tecnologías: Express.js, Stripe SDK

📦 Order Service (Puerto 3002)

Responsabilidades:

  • Gestión de pedidos
  • Estados de órdenes
  • Historial de compras
  • Integración con inventario

🔔 Notification Service (Puerto 3004)

Responsabilidades:

  • Envío de emails
  • Notificaciones push
  • SMS (opcional)
  • Templates de mensajes

📊 Admin Panel (Puerto 3021)

Responsabilidades:

  • Dashboard de administración
  • Gestión de productos y categorías
  • Reportes y analytics
  • Configuración del sistema

🔍 Solución de Problemas

❌ Error: EADDRINUSE (Puerto en uso)

Síntoma: Un servicio no puede iniciar porque el puerto está ocupado.

Solución:

# Encontrar proceso en puerto 3000
lsof -i :3000

# Matar proceso
kill -9 <PID>

# O matar todos los procesos Node
pkill -f "node.*service"

❌ Tests Fallando

Síntoma: Los tests de integración fallan.

Solución:

# Asegúrate de que los servicios estén corriendo
npm run dev:all

# Ejecuta solo unit tests
npm run test:unit

# Limpia cache de Jest
npm test -- --clearCache

❌ Links Rotos en el Frontend

Síntoma: Navegación entre páginas no funciona.

Solución:

# Validar todos los enlaces
npm run links:validate

# Ver reporte detallado
cat reports/link-validation-*.md
